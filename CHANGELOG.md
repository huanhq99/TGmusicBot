## [1.13.9] - 2026-03-14
### 修复 (Critical Fixes)
- **Emby 服务修复**: 修复了由于 API 调用参数类型错误导致的 `list object is not a mapping` 崩溃问题。这发生在设置歌单可见性时。
- **配置持久化增强**: 进一步巩固了 v1.13.8 的类型转换逻辑，确保 Web 端与机器人端配置读取绝对同步且具备极强的容错能力。
- **全线同步**: 统一所有文件的版本号为 v1.13.9。

## [1.13.8] - 2026-03-14
### 修复 (Critical Fixes)
- **修复 500 错误**: 修复了当扫描间隔等整数配置为空时，保存逻辑将其误转为字符串 `'false'` 导致加载页面时 `int()` 转换崩溃的问题。
- **增强类型安全**: 引入了 `safe_int` 转换机制，即使数据库中存在脏数据或空值，Web 界面和机器人也能正常启动并回退到默认值。
- **优化保存逻辑**: 改进了布尔值映射算法，仅对已知布尔键进行“真/假”强制转换，非布尔字段（如目录、模板、间隔）现在完美保留原始输入。

## [1.13.7] - 2026-03-14
### 修复 (Critical Fixes)
- **布尔值兼容性修复**: 修复了浏览器由于 `on` 状态发送导致的布尔值（自动下载、自动整理）在保存后无法被正确识别为 `True` 的严重 Bug。现在全面支持 `true`, `on`, `1`, `yes` 映射。
- **强制绝对路径**: 数据库路径现强制解析为绝对路径，彻底消除 `bot/bot.db` 与 `data/bot.db` 的路径冲突问题。
- **并发性能**: 启用了 SQLite WAL (Write-Ahead Logging) 模式，解决 Web 端与机器人端由于并发访问导致的数据写入失败。

## [1.13.6] - 2026-03-14
### 修复 (Critical Fixes)
- **设置持久化定稿**: 彻底统一了 Web 后端 (`web.py`)、前端 (`settings.html`) 和机器人进程 (`main.py`) 的配置键名（如 `organize_dir`）。
- **修复刷新失效**: 优化了设置保存逻辑，确保所有表单字段（包括勾选框和目录路径）都能正确存入数据库并支持刷新后实时加载。
- **脱离机器人依赖**: 设置的保存和读取现在完全基于数据库，不再依赖任何同步任务或进程间通信，彻底解决“保存成功但刷新消失”的逻辑 Bug。

## [1.13.5] - 2026-03-14
### 修复 (Critical Fixes)
- **配置保存极致简化**: 网页端保存设置现在仅负责写入数据库，不再尝试跨进程触发机器人状态、不再导入多余模块。彻底解决“保存刷新后消失”和“保存报错”的问题。
- **移除冗余同步**: 删除了 `config_sync_job` 轮询任务。机器人仅在需要配置时实时查库，减少日志噪音和系统开销。

## [1.13.4] - 2026-03-14
### 修复 (Critical Fixes)
- **同步降噪**: 优化 `config_sync_job` 逻辑，彻底修复每次扫描都误报“检测到配置变更”的问题。
- **稳定性增强**: 设置同步间隔调整为 30s，并增加了 `sqlite3` 连接的超时容错（15s）。

## [1.13.3] - 2026-03-14
### 修复 (Critical Fixes)
- **跨进程同步**: 彻底修复 Web 界面保存设置后，Bot 进程无法实时读取的问题。新增 `config_sync_job` 每 30 秒自动同步数据库配置。
- **持久化可靠性**: `web.py` 核心配置获取逻辑重构，绕过内存缓存直接读取数据库，解决“保存并刷新后消失”的问题。
- **Cookie 稳定性**: 为 `get_ncm_cookie` 和 `get_qq_cookie` 增加了独立数据库连接支持，确保在高并发下载时认证信息的进程安全性。
- **文件整理器**: 修复自动整理开关在 Web 端开启后，Bot 进程无需重启即可自动加载监控任务。
- **UI 清理**: 根据用户反馈，完全移除了网易云扫码登录（QR Login）相关残留逻辑和 UI。
